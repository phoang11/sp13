Hands-On Exercise 3.2
Automating the Time-Off Request Process
Exercise list …








Objective
The Time Off content type and list really helped us organize our vacation requests. Now we want to apply some automation to our current manager approval process.

In this exercise, you will

  • Build a simple manager approval workflow for time-off requests
Creating the time-off request workflow
1.
Open your department site in SharePoint Designer 2013 if it is not already open.
2.
Select Workflows from the Site Objects pane.
3.
Select the List Workflow button in the ribbon and click Time Off Requests from the menu.
4.

In the Create List Workflow – Time Off Requests dialog, enter the following:

 

Name: Time Off Request Process

Description: Time Off manager approval workflow

Platform Type: SharePoint 2013 Workflow

5.
Click OK.
6.
Click Time Off Request Process in the Designer breadcrumb trail.
7.
In the Settings area on the Time Off Request Process page, click the down arrow of the Task List pull-down menu.
8.
On the pull-down menu, select New Task List.
9.
A warning dialog will appear. Click OK.
10.
Click Yes on the second warning dialog.
11.
In the Start Options section, select the checkbox next to Start workflow automatically when an item is created.
12.
Save your work by clicking Save in the ribbon.
13.
Click Edit workflow in the Customization section.
14.
Click the words Stage 1 in the first stage of the workflow.
15.
Replace the stage name, Stage 1, with Time Off Approval and press <Enter>.
16.
Click just below the phrase "Start typing. . ." so that the orange blinking line appears. This is where you will insert your first action.
17.
Click the Action button in the ribbon and select Add Time to Date.
18.
Change the number of hours to 24 by clicking the 0 (zero) next to the word "hours" and typing 24
19.
For the Date value, click the word date, then click the fx button.
20.
In the "Lookup for Date/Time" dialog, set the data source to Current Item.
21.
Set "Field from source" to Created.
22.
Click OK.
23.
Click the link Variable: date.
24.
Select Create a new variable.
25.
In the Edit Variable dialog, set the name of the new variable to Tomorrow
26.
Click OK.
27.
Place the orange cursor below the action you just created.
28.
Add an Assign a task action.
29.
Click the this user link.
30.
In the "Assign a Task" dialog, click the ellipsis next to the Participant field.
31.
In the Select User dialog, select User who created current item from the list of users and groups.
32.
Click the Add button.
33.
Click OK.
34.

In the Task Title field, type the following:

 

Time Off Request: Please Review

35.
For the description, click Open editor for body to open the String Builder.
36.
Type the following: Please review the time off request from <Space>
37.
Click the Add or Change Lookup button.
38.

Set the fields to the following in the "Lookup for String" dialog box:

 

Data source: Current Item

Field from source: Created By

Return field as: Display Name

39.
Click OK in the "Lookup for String" dialog box. Your String Builder dialog should now look like this:
40.
Click OK to close the String Builder.
41.
For the task Due Date field, click the fx button.
42.
Set the Data Source to Workflow Variables and Parameters.
43.
Then set the "Field from source" pull-down to Variable: Tomorrow. Your Lookup dialog should look like this:
44.
Click OK to close the Lookup dialog.
45.
Click OK again.
46.
Finish the stage by placing the orange cursor in the "Transition to stage" section.
47.
Add a Go to a stage action.
48.
Click the a stage link in the "Go to a stage" action. Set the value to End of Workflow.
Your Time Off Approval stage should now look like this:
Perform a quick test of the workflow
49.
Publish the workflow by clicking Publish in the Save section of the ribbon.
50.
In the Site Objects pane, select List and Libraries.
51.
Click just to the right of the words Time Off Request to select (but not open) the Time Off Requests list.
52.
Select the Preview in Browser button to open the list in Internet Explorer.
53.
Create a new Time Off Request by clicking the new item link.
54.
Enter any number of hours for Time Off Hours. Then enter a Start Date and End Date. At this point in the workflow, we have no logic or conditions, so it doesn't matter what values you enter for this test.
55.
Click Save.
56.
Within 1 minute, you will get an e-mail from SharePoint. Open Outlook 2013.
57.
In the body of the new e-mail, there will be a link labeled Time Off Request: Please Review. Click that link.
58.
Internet Explorer will open to the Workflow Task. Click Edit Item in the ribbon.
59.
Approve the request by clicking Approved.
Congratulations! The workflow is now complete! This is a very simple approval workflow. In the next exercise, you will add conditions and actions to make it more realistic.

This is the end of the exercise.
