Hands-On Exercise 3.5
Modifying and Testing a Workflow
Exercise list …








Objectives

 

In this exercise, you will

  • Modify an existing stage to check for requests with over 40 hours
  • Test the updated workflow logic
Adding logic to check if request is over 40 hours
The other business rule to enforce in the workflow will check to see if the total hours requested is greater than 40. If so, HR needs to review the request.
1.
In the Time Off Approval stage, immediately after the If statement in the "Transition to stage" area, delete the line Go to Request Approved.
Hint ...
To delete an action or condition, highlight it and press the <Delete> key.
2.
Replace the deleted "Go to stage" action with an If any value equals value action.
3.
In the new If condition, set the first value to the following:
Data source: Current Item
Field from source: Time Off Hours
4.
Click OK.
5.
Change equals to is greater than.
6.
Set the second value to 40 and press the <Enter> key.
7.
Place the orange cursor beneath the If statement.
8.
Add a Go to a stage action.
9.
Set the stage to HR Review.
10.
Place the orange cursor in the Else branch.
11.
Add the action Go to a stage.
12.
Set the stage to Request Approved.
Your Time Off Approval stage should look like this:
13.
Publish your workflow.
14.
Right-click the Time Off Request tab in Designer.
15.
Select Close All Tabs from the menu.
Testing the updated Time Off Request Process
16.
In Designer, select Lists and Libraries from the Site Objects pane.
17.
Click Time Off Request from the Lists area.
18.
Select Preview in Browser from the ribbon.
19.
Click new item.
20.
Enter the following values for the time-off request:

Time Off Type: Vacation

Time Off Hours: 80

Time Off Comment: Two-week vacation

Time Off Start Date: [The next Monday]

Time Off Return Date: [Two weeks from next Monday]

21.
Click Save to submit the request.
22.
Refresh the Time Off Requests list in Internet Explorer.
The Time Off Request Process column of the Time Off Requests list will have the value Time Off Approval.
23.
Click the Time Off Approval link.
You will be taken to the Workflow Status page for the Time Off Request Process.
24.
Review the Workflow History. The first log message from the workflow has been added to the history.
25.
Close Internet Explorer.
26.
Go to your inbox in Outlook.
You should have received a new e-mail requesting that you complete a Time Off Request task.
27.
Click the Time Off Request: Please Review link in the e-mail.
Internet Explorer opens to the Time Off Request task.
28.
Click Edit Item from the ribbon.
29.
On the task assignment page, click Approved.
You will be taken to the Time Off Request Process Task list.
30.
Click Time Off Requests in the Quick Launch menu.
Notice that the Time Off Request Process column now reads HR Review.
31.
Close Internet Explorer.
32.
Open your inbox in Outlook.
33.
A new e-mail task has been sent to you.
34.
Click the Time Off Request: HR Approval Required link.
Internet Explorer will open to the Time Off Request task page.
35.
Click Edit Item in the ribbon.
36.
Click Approved on the Task page.
37.
Click Time Off Requests in the Quick Launch menu.
The Time Off Request Process column will now read Request Approved.
38.
Switch back to your Outlook inbox.
There will be a final e-mail from this test request stating that the time-off request has been approved.
Create another test in which the total hours requested is equal to 1.
Congratulations! You have updated the time-off request process with MSC time-off business rules.

This is the end of the exercise.
