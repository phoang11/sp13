Objective

 

We expect the Time Off Request process to be a well-used one. The CEO wants "one button" to start the process going. What the CEO wants, the CEO gets.

 

We'll meet the CEO's requirement for a Time Off Request button using a SharePoint Quick Step.

 

In this exercise, you will

  • Build a Quick Step button to trigger the time-off request approval process
Building a Quick Step button
1.
Open your Time Off Request list in Internet Explorer.
2.
Click the LIST tab in the ribbon.
3.
Click New Quick Step in the Customize List group of the ribbon.
4.
A warning dialog will appear. Click Allow.
SharePoint Designer will open with the "Add a Button" dialog.
5.
Select the Start an existing workflow radio button.
6.
Set the value of the Button Label to Time Off
7.

Set the Button Image value to the following URL:

 

http://home.msc.com/Site%20Images/QStep.jpg

8.
Click OK.
The Time Off Request Process workflow will open in Designer.
9.
Click Publish in the ribbon to publish the workflow.
10.
Select Lists and Libraries from the Site Objects menu.
11.
Click Time Off Requests from the list.
12.
Select Preview in Browser from the ribbon.
13.
Select a time-off request.
14.
Click the ITEMS tab in the ribbon.
The new Quick Step to start a time-off request is now visible in the ribbon. Do not click this button!
Moving the Time Off Quick Step to the Workflow ribbon group
15.
Open the Time Off Requests library in Designer.
16.
Click Time Off in the Custom Actions section.
17.
Scroll to the bottom half of the Create Custom Action dialog so that the Advanced custom action options are visible.
18.

Change the Ribbon Location field to the following:

 

Ribbon.ListItem.Workflow.Controls._children

19.
Change the sequence number to 100. The dialog should look like this:
20.
Click OK.
Changing the Time Off Request workflow start option
21.
Click Time Off Request Process in the Workflow section.
22.
Clear the box for "Start workflow automatically when an item is created" in the Start Options section.
Time-off requests will now begin when the Quick Step is clicked.
23.
Publish the Time Off Request Process.
24.
To test, go to the Time Off Request list in Internet Explorer.
25.
Select the checkbox on the left side of an existing request, and click the Time Off Request button in the ribbon. This will relaunch a previous request.
26.
Once you have test-launched a Time Off Request workflow, go back to SharePoint 2013 Designer. Right-click any tab in Designer.
27.
Select Close All Tabs.
Why disable the auto-start process for the workflow? Doing so gives the employees of MSC control over when they submit their requests. It also allows them to submit requests in batches as needed.
Change the behavior of the Time Off Quick Step from starting the workflow to creating a new Time Off Request when clicked.

This is the end of the exercise.
