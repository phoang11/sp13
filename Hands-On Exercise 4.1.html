Hands-On Exercise 4.1
Editing Workflows With Visual Designer
Exercise list …








Objectives

 

The HR department has requested that the Time Off Request process log each of the transitions.

 

SharePoint Designer provides an alternate view of our workflows called Visual Designer.

 

In this exercise, you will

  • Use Visual Designer to add actions to log events to the workflow history
  • Add an approved time-off request to the department calendar
Adding log actions using Visual Designer
1.
Open your department site in SharePoint Designer 2013 if it is not already open.
2.
From the Site Objects pane, select Workflows.
3.
Open the Time Off Request Process, if it's not already open.
4.
In the Customization section of the Designer page, click Edit workflow.
5.
In the Manage group of the ribbon, click the Views button.
Scroll around Visual Designer to familiarize yourself with the location of each workflow stage.
6.
From the Shapes stencil menu, select the Actions – SharePoint 2013 Workflow.
7.
In the Shapes panel, locate the "Log to history list" action.
The shapes are listed in alphabetical order.
8.
Click and drag the Log to history list shape to the space between the green start arrow and the "Add time to date" action of the Time Off Approval stage.
Hint ...
The connector lines (connecting shapes to one another) will automatically bind to the "Log to history list" shape.
The Time Off Approval stage will now look similar to this:
9.
If not already selected, click the Log to history list shape to select it.
The Settings icon will appear in the lower left corner of the shape.
10.
Click the Settings icon to open the Settings menu.
11.
Select Message from the menu.
12.
Type the following into the message field of the Log to History List Properties dialog:
The time off request meets the minimum hours policy.
13.
Click OK.
14.
Drag and drop a Log to history list to the HR Review stage. Place it to the left of the "Assign a task" stage, like this:
15.
If not already selected, click the Log to history list shape to select it.
16.
Click the Settings icon to open the Settings menu.
17.
Select Message from the menu.
18.
Type the following into the Message field of the "Log to History List Properties" dialog:
The time off request exceeds 40 hours and requires HR approval.
19.
Click OK.
20.
Add a Log to history list action to the Request Approved stage to the left of the "Send an email" shape:
21.
Type the following into the message field of the "Log to History List Properties" dialog:
The time off request was approved.
22.
Finally, add a Log to history list action to the Request Rejected stage to the left of the "Send an email" shape:
23.
Type the following into the message field of the "Log to History List Properties" dialog:
The time off request was rejected.
24.
Click the Publish button to publish the workflow.
25.
Click the Views button in the ribbon to return to Text-Based Designer mode.
Adding an Approved Time Off request to the department calendar
26.
In Designer, from the Site Objects pane, click Lists and Libraries.
27.
Click SharePoint List from the New group in the ribbon.
28.
Select Calendar from the menu.
29.
Enter the following in the "Create list or document library" dialog:
Name: [Department] Calendar
30.
Click OK.
31.
From the Site Objects pane, select Workflows.
32.
Open the Time Off Request Process workflow, if not already open.
33.
In the Customization section, click Edit workflow.
34.
Click the Views button in the ribbon, to return to the Visual Designer.
35.
Drag and drop the Create list item to the right of the "Send an email" shape in the Request Approved stage.
36.
Click the Settings icon of the "Create list item" shape.
37.
Select Item from the menu.
38.
From the List pull-down of the Create New List Item dialog, select [Department] Calendar.
39.
Click the Add button.
40.
In the "Set this field" pull-down, select Title.
41.
Click the ellipse button in the "To this value" field.
42.
Click the Add or Change Lookup button in the String Builder dialog.
43.
Set the following values in the Lookup for Single dialog:
Data source: Current Item
Field from source: Created By
Return field as: Display Name
44.
Click OK.
45.
Type the following:
<Space> is out of the office
46.
Click OK to close the String Builder.
47.
Click OK again to close the Value Assignment dialog.
48.
Select the Start Time field.
49.
Click the Modify button.
50.
Click the fx button for the "To this value" field.
51.
Set the following values in the "Lookup for Date and Time" dialog:
Data source: Current Item
Field from source: Time Off Start Date
52.
Click OK.
53.
Click OK again to close the Value Assignment dialog.
54.
Select the End Time field.
55.
Click the Modify button.
56.
Click the fx button for the "To this value" field.
57.
Set the following values in the "Lookup for Date and Time" dialog:
Data source: Current Item
Field from source: Time Off Return Date
58.
Click OK.
59.
Click OK again to close the Value Assignment dialog.
The Create New List Item dialog should look like this:
60.
Click OK to close the Create New List Item dialog.
61.
Click the Publish button in the ribbon to publish the workflow.
62.
Click the Views button in the ribbon to return to the Text-Based Designer view.
Testing the updated workflow
63.
From the Site Objects pane, select Lists and Libraries.
64.
Open the Time Off Requests list.
65.
Click the Preview in Browser button in the ribbon.
66.
In Internet Explorer, select the first Time Off Request item created before a workflow was added to the list. It is the Long Weekend one.
67.
Open the ITEMS tab in the ribbon.
68.
Click the Workflows button in the ribbon.
The Time Off Requests: Workflows page will open.
69.
In the "Start a New Workflow" section, click the Time Off Request Process link.
Internet Explorer will return to the Time Off Requests list.
70.
Refresh the page.
71.
Open the inbox in Outlook.
You will get an e-mail from the time-off request workflow.
72.
Click the Time Off Request: Please Review link in the e-mail.
73.
Click Edit Item on the Time Off Request task page in Internet Explorer.
74.
Click Approved.
75.
Select Time Off Requests from the Quick Launch.
76.
Click the Request Approved link in the Time Off Request Process column.
The workflow status page will open. Notice that there are now three entries in the workflow history. Logging to the workflow history is a useful way to track progress and debug workflows.
77.
Click the [Department] Calendar link in the Quick Launch.
You will see that the time-off request has been added to the department calendar.
Congratulations! You have used Visual Designer to connect a SharePoint 2013 workflow to other SharePoint lists.

This is the end of the exercise.
