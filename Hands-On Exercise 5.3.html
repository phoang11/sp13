Hands-On Exercise 5.3
Workflow Recall With Parallel Processes
Exercise list …








Objectives

 

MSC needs an update to the Internal Project Budget review process. Budget submitters need a way to recall the budget during the manager review cycle.

 

In this exercise, you will

  • Create a budget recall process using a parallel workflow
  • Test the workflow
Creating a budget recall process using a parallel workflow
1.
Open your Internal Project Budget Review workflow in Designer if it is not already open.
2.
Click Edit Workflow in the Customization section of the page.
3.
Place the orange cursor above the Budget Review step.
4.
Click Parallel Block in the Insert group of the ribbon.
5.
Select the Budget Review step.
6.
Right-click the Step header and select Move Up.
The entire Budget Review step is moved into the parallel block.
7.
Right-click the top of the parallel block.
8.
Select Advanced Properties from the menu.
9.
In the Properties dialog, set the Completion Condition to Create a new variable.
10.
Set the Name field to CanBeRecalled in the Edit Variable dialog.
11.
Click OK.
12.
Click OK on the Properties dialog.
13.
Place the orange cursor above the Assign a task action.
14.
Add a Set Workflow Variable action.
15.
Click the workflow variable link.
16.
Select Variable: CanBeRecalled from the menu.
17.
Click the value link.
18.
Select Yes from the pull-down menu.
Setting the CanBeRecalled variable to Yes tells Designer that if any of the steps in the parallel process terminate (Rejected/Recalled), all of the steps terminate.
19.
Place the orange cursor below the Budget Review step.
20.
Click Step in the ribbon.
21.
Name the new step Recall Process
22.
Place the orange cursor in the Recall Process step.
23.
Add an Assign a Task action.
24.
Click the this user link.
25.
In the "Assign a Task" dialog, set the following:

Participant: [User who created current item]

Task Title: Internal Project Budget: Recall Budget

Description: If you need to recall/cancel your project budget edit this task and click the Recall button.

26.
Click the inverted chevron next to the words Outcome Options.
27.
Change the "Task content type" to Project Document Workflow Task.
28.
Set the Default outcome to Complete.
29.
Click OK.
30.
Click the Variable: Outcome link in the "Assign a Task" action.
31.
Select Create a new variable from the pull-down menu.
32.
Set the Name value to Recall Outcome
33.
Click OK.
34.
Click Save to save your work.
35.
Place the orange cursor below the Assign a Task action in the Recall Process step.
36.
Add a Log to History List action.
37.
Click the message link in the "Log to History List" action.
38.
Set the message to:
The internal project budget has been canceled.
39.
Click Save to save your work and click Publish to publish the workflow.
40.
Go to the Internal Project Budget Review workflow summary page of the Internal Project Budget Review workflow.
41.
Click the Open Task List link.
42.
Set the default content type to Workflow Task (SP2013).
43.
In the ribbon, click Publish to publish the workflow.
Testing the workflow
44.
Click Lists and Libraries in the Site Objects pane.
45.
Click Internal Project Documents in the Document Libraries section.
46.
Click Preview in Browser in the ribbon.
The Internal Projects Documents section will open in Internet Explorer.
47.
Click FILES in the ribbon tabs.
48.
Click New Document in the New group of the ribbon.
49.
Click Yes in the Microsoft Office warning dialog.
Microsoft Word 2013 opens with a new Internal Project Budget document.
50.
Enter the following test data in the Internal Project Budget Properties pane:

Title: MSC North Shore Project Budget

Project Name: North Shore Project

Project Cost: 50000

Project Start Date: [Enter tomorrow's date]

Project End Date: [Enter a date 6 months in the future]
Project Budget Due Date: [Enter the date of next Monday]

51.
In the document, enter We need $50,000 for Environmental cleanup after drilling, and click Save.
52.
Select Other Web Locations in the Save As menu.
53.
Click the Browse button.
54.
Name the file MSC North Shore Project Budget in the Save As dialog, then click Save.
55.
Close Microsoft Word.
56.
Refresh the Internal Projects Document page in Internet Explorer.
If you get the following warning message, click Retry.
57.
Click the Manager Review Cycle link in the Internal Project Budget Review column.
It may take up to a minute before the Manager Review Cycle is visible in the Internal Project Budget Review column.
Verify that the workflow has started.
58.
Open your inbox in Outlook.
You will receive two e-mails since you are playing both the submitter and the manager role.
59.
Open the e-mail with the subject Task Assigned – Internal Project Budget: Manager Review Requested.
60.
Click the Internal Project Budget: Manager Review Requested link.
The Manager Review task will open in Internet Explorer.
61.
Click Edit Item in the Manage group of the ribbon.
62.
Click Rejected at the bottom of the page.
If the Content Type field is set to Project Document Workflow Task, change it to Workflow Task (SharePoint 2013).
63.
Close Internet Explorer.
64.
Open the new e-mail in Outlook with the subject Task Assigned – Internal Project Budget: Update Requested by Manager.
It may take up to a minute to receive this e-mail (usually less than a minute).
65.
Click the link Internal Project Budget: Update Requested by Manager.
66.
Click Edit Item in the ribbon in Internet Explorer.
67.
Click Complete.
68.
Close Internet Explorer.
69.
Open the e-mail in Outlook with the subject Task Assigned – Internal Project Budget: Recall Budget.
70.
Click the link Internal Project Budget: Recall Budget.
71.
Click Edit Item in the ribbon in Internet Explorer.
72.
Click the Recall button.
By clicking Recall, the Budget Review Loop was canceled, and both paths of the parallel process were canceled.
73.
Still in Internet Explorer, click Internal Project Documents in the Quick Launch.
74.
Click the Manager Review Cycle link.
The Internal Status (workflow status) shows Completed.
The last entry in the log reads, "The internal project budget has been canceled."
A new e-mail appears in Outlook:

If your recall fails to halt the workflow and set its status to Completed, perform the following check.

a. Open the workflow in Text Designer.

b. Repeat steps 7 through 12 in this exercise.

c. Retest your workflow.

Congratulations! You have added a parallel workflow process with the ability for the submitter to cancel the workflow mid-flight!

This is the end of the exercise.
