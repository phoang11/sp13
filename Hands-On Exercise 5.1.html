Hands-On Exercise 5.1
Budget Approval Process
Exercise list …








Objectives

 

The chief financial officer (CFO) is enforcing policies to control spending on internal MSC projects, like opening a new location, launching a marketing campaign, or acquiring other consulting companies.

 

To support the CFO, each department has been tasked with implementing an MSC Project Budget Approval process.

 

The built-in workflow task content type has two choices: Approve and Reject. We want to not just approve documents but also to mark them as Complete or Incomplete.

 

To do this, we will need a custom 2013 workflow approval content type.

 

In this exercise, you will

  • Create the Internal Project Budget Content Type
  • Use an Internal Project Budget content type and library
  • Add a custom workflow approval content type
  • Create a workflow task content type
Create the Internal Project Budget Content Type
In the Time Off Approval workflow, we used a list content type. In this exercise, we will create a document content type for our Budget Approval Process
1.
Open your department site in SharePoint Designer 2013.
2.
Click Site Columns in the Site Objects pane.
3.
Create the first site column by clicking New Column in the New section of the ribbon and selecting Single line of text.
4.

Enter the following in the "Create a Site Column" dialog box and click OK:

 

Name: Project Name

New Group: _[Department]

 

Click OK.

5.
Save your work by clicking the Save button or pressing <Ctrl><S>.
6.

Add the following site columns and add them to the existing site column group _[Department]:

 

NameType
Project CostCurrency
Project Start DateDate & Time
Project End DateDate & Time
Project Budget Due DateDate & Time
Project Business CaseYes/No
7.
Click Save to save your work.
8.
Select Content Types from the Site Objects pane.
9.
Click Content Type in the New section of the ribbon to create a new content type.
10.

In the "Create a Content Type" dialog box, enter the following:

 

Name: Internal Project Budget
Select parent content type from: Document Content Types

Select parent content type: Document

New group: _[Department]

 

Click OK.

11.
Click the Internal Project Budget content type you just created in the Content Types list.
12.
On the Internal Project Budget content type tab in Designer, click Edit content type columns in the Customization section.
13.
Click Add Existing Site Column from the New section of the ribbon.
14.
From the Site Column Picker dialog, double-click the Project Name site column.
15.

Add the remaining Time Off columns by repeating the above steps in the following order:

 

Project Cost

Project Start Date

Project End Date

Project Budget Due Date

Project Business Case

If your Add Existing Site Column button is disabled in the ribbon, click anywhere in the Columns editor.
16.
Click Save to save your work.
Use an Internal Project Budget content type and library
17.
Select Lists and Libraries from the Site Objects pane.
18.
Click Document Library in the New group of the ribbon.
19.
Select Document Library from the menu.
20.
Name the library Internal Projects Documents
21.
Click OK.
22.
Click the Internal Projects Documents library from the Document Libraries list.
23.
Select the checkbox next to Display this list on the Quick Launch in the Settings section.
24.
Select the box next to Allow management of content types in the Settings section.
25.
Click Save to save your work.
26.
Click the Add button in the Content Types section of the library settings page.
27.
Select the Internal Project Budget from the Content Type Picker dialog.
The Internal Project Budget content type is located in the MSC content type group.
28.
Click OK.
29.
Select the Document content type by clicking to the right of the word "Document."
30.
Click the Delete button in the ribbon.
31.
A Confirm Delete dialog will open. Click Yes.
Adding a custom workflow approval content type
32.
In SharePoint Designer, select Site Columns from the Site Objects pane.
33.
Click New Column from the ribbon.
34.
Choose Task Outcome from the menu.
35.
In the "Create a Site Column" dialog, set the following values:

Name: Project Document Task Outcome

Group: _[Department]

36.
Click OK.
The Column Editor dialog will open.
37.
In the Column Editor dialog, replace the Choices values with the following:

Choices: Incomplete

         Complete
         Recall

38.
Set the "Default value" to Incomplete
39.
Clear the box next to "Allow blank values?".
40.
Click OK.
41.
Click Save to save your work.
Creating a workflow task content type
42.
Select Content Types from the Site Objects pane.
43.
Click Content Type in the new group of the ribbon.
44.
Set the following values in the "Create a Content Type" dialog.

Name: Project Document Workflow Task

Parent content type from: List Content Types

Parent content type: Workflow Task (SharePoint 2013)

Existing group: _[Department]

45.
Click OK.
46.
Select the Project Document Workflow Task from the Content Types list.
The Project Documents Workflow Task tab will open in Designer.
47.
Click Edit content type columns in the Customization section of the page.
48.
Select the Task Outcome column.
49.
Click the Delete button in the Edit group of the ribbon.
50.
Click the Save button to save your work.
51.
Click Add Existing Site Column.
52.
In the Site Column Picker dialog, select Project Document Task Outcome.
53.
Click OK.
54.
Click Save to save your work.
55.
Right-click the Project Document Workflow Task tab.
56.
Select Close All Tabs from the menu.
Congratulations! You have created a new type of content type available in SharePoint 2013 for customized workflow task outcomes.

This is the end of the exercise.
