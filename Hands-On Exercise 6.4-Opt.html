Optional Hands-On Exercise 6.4
Connecting a Workflow to a Web Service
Exercise list …








Objective
Employees have asked for a weather service app on their department home pages. We are going to create a workflow to get weather information for our island from a web service and display it on the home page.

In this exercise, you will

  • Create a Local Weather list for the five-day forecast
  • Build a workflow to run each day that gets the local five-day forecast from a weather web service
Creating the Local Weather lists
1.
Open your department site in SharePoint Designer 2013 if it is not already open.
http://[island].msc.com/sites/[department]
2.
From the Lists and Libraries in Site Objects, click SharePoint List in the ribbon and select Promoted Links.
3.
Name the list Local Weather and click OK.
This list will hold our weather information from the web service.
4.
Click the Local Weather link to open the list for editing.
5.
Click Edit list columns link and click the Description row to select the Description column.
6.
Click Column Settings in the ribbon, select Rich Text (Bold, ...) and Enhanced Rich Text. Click OK.
7.
Save your work.
Removing the "Get started with your site" Web Part and replacing it with the Local Weather Web Part
8.
If not already open, open Internet Explorer and go to your department's home page.
http://[island].msc.com/sites/[department]
9.
Click EDIT in the upper right corner.
10.
Delete the "Get started with your site" Web Part by clicking REMOVE THIS next to the "Get started with your site" title.
We will replace this list with our insert, the Local Weather list Web Part.
11.
Place the cursor in the top zone (box) on the page.
12.
Click the INSERT tab in the ribbon. Click App Part.
13.
Select Local Weather and click the Add button at the right of the screen.
This will place an empty promoted list app on the page.
14.
Click Save in the ribbon.
Building weather workflow
15.
Return to SharePoint Designer.
16.
Select Workflows from the Site Objects pane.
17.
Click the Site Workflow button in the ribbon.
18.
Name the workflow Get Local Weather
19.
Click OK.
20.
Rename the first stage Initialize Variables
21.
Place the orange cursor within the Initialize Variables stage.
22.
Add a Set Workflow Variable action.
23.
Click the workflow variable link.
24.
Select Create a new variable from the menu.
25.

Set the following values in the Edit Variable dialog:

 

Name: Index

Type: Integer

26.
Click OK.
27.
Select the value link.
28.
Type 0 as the value and press <Enter>.
29.
Place the orange cursor beneath the Set Workflow Variable action.
30.
Add another Set Workflow Variable action.
31.
Click the workflow variable link.
32.
Select Create a new variable from the menu.
33.

Set the following values in the Edit Variable dialog:

 

Name: Region

Type: String

34.
Click OK.
35.
Select the value link.
36.
Type your island name (e.g., Hawaii).
37.
Place the orange cursor below the Initialize Variables stage.
38.
Add a new stage and name it Get Weather Service
39.
Place the orange cursor in the Transition to stage section of the Initialize Variables stage.
40.
Add a Go to a Stage action.
41.
Set the "Go to a Stage" action to Get Weather Service.
42.
Place the orange cursor in the Get Weather Service stage.
43.
Add a Call HTTP Web Service action.
44.
Click the this link.
45.
Open the Course Files folder at C:\Course Files.
46.
Open the WebServiceURL.txt file.
47.
Copy and paste the content of the file into the HTTP web service URL field.
48.
Replace the [Your Region Here] text with your island (e.g., Hawaii).
49.
Click OK.
50.
Select the response link.
51.
Click Create a new variable.
52.
Set the name to Weather Content in the Edit Variable dialog.
53.
Click OK.
54.
Below the Call Action, add a Count Items in Dictionary action.
55.

In the "Count Items in Dictionary" action, set these values:

 

Dictionary: Variable: Weather Content

Output Variable: Item Count (new Integer variable)

56.
Place the orange cursor below the Count Items in Dictionary action.
57.
Add a Loop n Times container.
58.
Click the 1 link.
59.

The LoopNTimes Properties dialog opens. Set the following values:

 

Title: Forecast Loop

RepeatCount: Variable: Item Count

60.
Click OK.
61.
In the Forecast Loop, add a Get an Item from a Dictionary action.
62.

Set the action properties:

 

Item path: ([%Variable: Index%])/Region

Click the ellipsis (...), click the "Add or Change" lookup button, and select workflow Variable: Index.
63.
Click the dictionary link.
64.
Select Weather Content from the menu.
65.
Click the item link.
66.
Select Create a new variable.
67.

Set the Edit Variable values:

 

Name: WxRegion

Type: String

Warning! All names related to getting information from the web service are case-sensitive.
68.
Add the Get an item by name or path from dictionary action for each of the following values. Do not forget the opening and closing parenthesis around the variable.
You can copy and paste the Get Item actions.

Day of Week

Item Path: ([%Variable: Index%])/DayOfWeek

Dictionary: Variable: Weather Content

Output Item: Variable: WxDay (new String variable)

High Temp

Item Path: ([%Variable: Index%])/FHigh

(If you prefer, you can use CHigh for Celsius)

Dictionary: Variable: Weather Content

Output Item: Variable: WxHigh (new String variable)

Low Temp

Item Path: ([%Variable: Index%])/FLow

(If you prefer, you can use CLow for Celsius)

Dictionary: Variable: Weather Content

Output Item: Variable: WxLow (new String variable)

Graphic URL

Item Path: ([%Variable: Index%])/GraphicURL

Dictionary: Variable: Weather Content

Output Item: Variable: WxURL (new String variable)

Summary

Item Path: ([%Variable: Index%])/Summary

Dictionary: Variable: Weather Content

Output Item: Variable: WxSummary (new String variable)

69.

Still in the Forecast Loop, create a new workflow variable for the weather description that will be displayed to the user.

 

Add a Set Workflow Variable action below the Get Item actions.

70.

Set the following value:

 

Workflow variable: WxDescription (new String variable)

71.
Click the value link and open the String Builder.
72.
Open the text file C:\Course Files\WxDescription.txt.
73.
Copy and paste the contents into the String Builder, but do not hit OK yet!
74.
Highlight and replace the Variables using the Add or Change Lookup button. Make sure you select everything between and including the opening and closing brackets.
When complete, the String Builder will look like this:
75.
Click OK.
76.
Place the orange cursor beneath the Set Variable: WxDescription action.
77.
Add a Create List Item.
78.

Click the this list link and set the following field values by selecting the field and clicking the Modify button:

 

This list: Local Weather

 

Fields:

Title: Variable: WxDay
Link Location Variable: WxURL

Launch Behavior: In page navigation

79.

You will need to click the Add button to add the remaining value pairs:

 

Order: Variable: Index
Background Image Location: Variable: WxURL
Description: Variable: WxDescription

80.
Click OK.
81.
Place the orange cursor beneath the Create List Item action.
82.
Add the action Do Calculation.
83.

Set these values:

Second value: 1

(new Number variable)

84.
Place the orange cursor beneath the Do Calculation action.

Add a Set Workflow Variable action. Set these values:

 

Workflow variable: Variable: Index
