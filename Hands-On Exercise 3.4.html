Hands-On Exercise 3.4
Using Workflow Conditions to Enforce Business Rules
Exercise list …








Objectives

The workflow should check to make sure that the requestor has requested at least two hours off.

 

For training purposes, you will play all roles in this workflow.

In this exercise, you will

  • Add an additional stage to the workflow for the HR review
  • Use conditions to enforce time-off–request business rules 
Inserting a stage for HR review
1.
Open your department site in SharePoint Designer 2013 if it is not already open.
2.
Click Workflows in the Site Objects pane.
3.
Select the Time Off Request Process.
4.
In the Customization section, select Edit workflow.
5.
Place the orange cursor between the Time Off Approval and Request Approved stages
6.
Add a new stage.
7.
Rename the new stage HR Review. Your workflow should look like this:

There are two ways to add actions and conditions:

  1. In the Insert group of the ribbon, click Action or Condition.
  2. Start typing the name of the action or condition where you would like to place it.
8.
Place the orange cursor in the HR Review stage.
9.
Type the word Assign. Designer will prompt you to press <Enter> to insert an "Assign a Task" action.
10.
In the "Assign a Task" action, click this user.
11.

The "Assign a Task" dialog will open. Set the following values:

 

Participant: User who created current item

Task Title: Time Off Request: HR Approval Required

Description: A time off request requires your approval.

12.
Click OK.
13.
In the "Assign a Task" action, click Variable: Outcome1.
14.
Change the outcome variable by selecting the pull-down and clicking Create a new variable.
15.
In the Edit Variable dialog, name the new variable HR Outcome and leave the Type set to Integer.
16.
Click OK.
17.
In the "Transition to stage" section of the HR Review stage, add an If value equals value condition.
Your HR Review stage should now look like this:
18.

Set the first value setting to the following:

 

Data source: Workflow Variables and Parameters

Field from source: Variable: HR Outcome

19.
Click OK.
20.
Set the second value setting to Approved.
21.
Once the HR Review is approved, direct the workflow to the Request Approved stage. To do this, place the orange cursor beneath the condition you created in the previous steps.
22.
Insert a Go to a stage action.
23.
Set the stage to Request Approved.
24.
Place the orange cursor below the word Else in the HR Review stage.
25.
Insert a Go to a stage action.
26.
Set the stage to Request Rejected.
The HR Review stage should look something like this (note: HR Result should be HR Outcome):
Using conditions to enforce time-off request business rules
27.
Place the orange cursor immediately above the Time Off Approval stage.
28.
Add a new stage.
29.
Rename the new stage Start Workflow
30.
Add a Log to History action.
31.
Click the message link.
32.
Open the String Builder by clicking the ellipsis button.
33.

Type the following in the String Builder:

 

Time Off Request started for <Space>

34.
Click the Add or Change Lookup button to open the "Lookup for String" dialog.
35.

In the "Lookup for String" dialog, set the following values:

 

Data source: Current Item

Field from source: Created By

Return field as: Display Name

36.
Click OK.
37.
Click OK again to close the String Builder.
38.
Place the orange cursor after the Log to History action.
39.
Add a Set Workflow Variable action.
40.
Click the workflow variable link in the Set Workflow Variable action.
41.
From the pull-down, select Create a new variable.
42.

In the Edit Variable dialog, set the following values:

 

Name: Department

Type: String

43.
Still in the Set Workflow Variable action, click the value link.
44.
Type your account name.
For example, if you are the Healthcare department on Hawaii, you would enter Hawaii.Healthcare as your account name.
45.
Place the orange cursor below the Start Workflow stage.
46.
Add a new stage.
47.
Name the new stage Minimum Hours Required
48.
Place the orange cursor in the Transition to stage section of the Start Workflow stage.
49.
Add an If any value equals value action.
50.

Change the first value to the following:

 

Data source: Current Item

Field from source: Time Off Hours

51.
Click OK.
52.
Change the equals to is greater than or equal to.
53.
Set the second value to 2
54.
Place the orange cursor beneath the If condition.
55.
Add a Go to a stage action.
56.
Set stage to Time Off Approval
57.
Place the orange cursor beneath the Else condition.
58.
Add a Go to a stage action.
59.
Set the stage to Minimum Hours Required
60.
Place the orange cursor in the Minimum Hours Required stage.
61.
Add a Log to History action.
62.

Set the log message to the following:

 

Time Off Request for <Space>

63.
Click the Add or Change Lookup button.
64.

In the "Lookup for String" dialog, set the following values:

 

Data source: Current Item

Field from source: Created By

Return field as: Display Name

65.
Click OK.
66.

Still in the String Builder, type the following:

 

<Space> did not meet the minimum of 2 hours.

67.
Click OK.
68.
Place the cursor below the Log message action.
69.
Add a Send an Email action.
70.
In the "Send an Email" action, click the these users link.
71.

In the Define E-Mail Message dialog, set the following values:

 

To: User who created current item

Subject: Time Off Request: Not Enough Hours

Body: Your time off request must be for at least 2 hours. Please correct and resubmit.

72.
Click OK.
73.
Place the orange cursor in the Transition to stage section of the Minimum Hours Required stage.
74.
Add a Go to a stage action.
75.
Set the value of "Go to a Stage" to End of Workflow
The beginning of your updated Time Off Request Workflow should look similar to this (note it's missing the first then statement):
76.
Click Publish in the ribbon to publish the workflow.
Test the updated Time Off Request workflow
77.
Open the Time Off Requests list in Internet Explorer.
78.
Create a new Time Off Request.
79.

Enter the following values in the new request:

 

Time Off Type: Vacation

Time Off Hours: 1

Time Off Start Date: [Today's Date]

Time Off Return Date: [Today's Date]

80.
Click Save.
You will receive an e-mail stating you have not requested enough hours.
Congratulations! You have saved the approving manager time by incorporating a simple business rule into the Time Off Request process.

This is the end of the exercise.
