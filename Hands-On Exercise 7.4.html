Hands-On Exercise 7.4
SharePoint Data Views

Objectives
Display customer work orders using the SharePoint Data View app.

In this exercise, you will

  • Link SharePoint to customer work orders
  • Use Data Views to create a master/detail customer work order site
Linking SharePoint to customer work orders
1.
Open your department site in Designer if it is not already open.
2.
Select External Content Types from the Site Objects pane.
3.
Click External Content Type in the ribbon.
4.

Set the following values in the External Content Type Information section:

 

Name: [Island][Department]WorkOrders
Display Name: [Island] [Department] Work Orders
Office Item Type: Task

Offline Sync for external list: Enabled

5.
Click the link to set the external data source.
6.
Expand the [Island] [Department] Database Tables folder.
7.
Right-click the CustomerWorkOrders table.
8.
Select Create All Operations from the menu.
9.
Click Next in the All Operations dialog.
10.
Select the OrderID field on the Parameters Configuration page.
11.
Check Map to Identifier in the Properties pane.
12.
Select the Task field.
13.
Change the Office Property to Subject in the Properties pane.
14.
Select the StartDate field.
15.
Change the Office Property to StartDate
16.
Select the DueDate field.
17.
Set the Office Property to DueDate
18.
Click Next.
19.
Click the Add Filter Parameter button.
20.
Add a filter parameter by clicking (Click to Add) in the Properties.
21.
Name the filter LimitFilter
22.
Change the filter type to Limit.
23.
Click OK.
24.
Set the Default Value to 2000
25.
Click Finish and Save.
26.
Click the Create Lists & Form button in the ribbon, and set the List Name to Work Orders in the Create List and Form dialog.
27.
Click OK.
28.
Open the Work Orders list in Internet Explorer.
There are almost 2,000 work orders listed. Later in the exercise, you will filter this down to only work orders related to your department's customers.
29.
Close Internet Explorer.
Creating a data view web page
30.
Select Site Pages from the Site Objects pane.
31.
Click the Web Part Page button in the New ribbon group.
32.
Select the top-left template.
33.
Set the page name to CustomerWork.aspx
34.
Right-click CustomerWork.aspx.
35.
Select Edit File in Advanced Mode from the menu.
The code view of CustomerWork.aspx will open in Designer:
36.
Locate the last closing <div> tag in the code view. Place your cursor immediately after the tag.
37.
Press <Enter>.
38.
Select the INSERT tab in the ribbon and click the HTML button in the Controls group of the ribbon.
39.
Select the <div> tag.
You should now have new opening and closing <div> tags.
40.
Click Save to save your work.
If a warning message appears, click Yes to close it.
41.
Place your cursor between the two <div> tags.
42.
Select the INSERT tab of the ribbon.
43.
Click the Data View button in the Data Views & Forms group of the ribbon.
44.
Scroll down the Data View menu and select Customers.
45.
Scroll down the page until you get to the <XmlDefinition> tag. Place your cursor anywhere within that tag body, such as immediately to the right of the <XmlDefinition> tag.
It is important that you activate the data view controls by placing the cursor within the data view code.
46.
Save your work.
47.
Click the Add/Remove Columns button in the ribbon.
48.

Remove all but the following fields in the Displayed Fields dialog:

  • BusinessEntityID
  • FirstName
  • LastName
  • PhoneNumber
  • EmailAddress
49.
Click OK.
You may have noticed the code change. The number for <ViewFields> has been reduced. We could edit the code directly, but using the ribbon commands saves time and reduces the risk of errors.
50.
Click the Paging button in the ribbon.
51.
Select Display Sets of 10 Items from the menu.

Here is the corresponding code:

 

<RowLimit Paged="TRUE">10</RowLimit>

52.
Select Sort & Group from the ribbon.
53.
Select BusinessEntityID from the "Available fields" list.
54.
Click the Add button to add it to the "Sort order" list.
55.
Click OK.
56.
Click Save.
Adding a related data view for work orders
57.
Find the closing <div> tag located at the end of the data view Web Part added in previous steps.
You can search for </div> by pressing <Ctrl><F>.
When searching for tags, click the Find All button to get a list of tag locations. In this case, we want the last </div> tag.
58.
Place the cursor just before the </div> tag.
59.
Select the INSERT ribbon tab.
60.
Click the Data View button in the ribbon.
61.
Select Work Orders from the menu.
62.
Place your cursor anywhere within the new data view, such as immediately to the right of the <XmlDefinition> tag.
63.
Click Add/Remove Columns from the ribbon.
64.
Remove BusinessEntityID from the list of Displayed Fields.
65.
Click OK.
66.
Select the Sort & Group button in the ribbon.
67.
Add StartDate to the "Sort order" list.
68.
Click OK.
Connecting the data views
69.
Place your cursor in the Customers data view.
Look for this segment of code to identify the Customers data view:
70.
Click Add Connection in the Connections group of the ribbon.
71.
Click Next in the Web Part Connections Wizard.
72.
Click Next again.
73.
Click Next.
74.
Set the first field in the second row for Columns in Customers to BusinessEntityID.
75.
Click Next.
76.
Click Finish.
77.
Click Save.
Reviewing your work
78.
Open the CustomerWork.aspx page in Internet Explorer.
79.
Click one of the diagonal double arrows in the top data view.
The related work order information will be displayed in the bottom data view.
Enhance the Data View Web Part presentation
80.
In SharePoint Designer, click inside the Customers data view in the CustomerWork.aspx file.
81.
Select the DESIGN tab.
82.
In the View Style section of the ribbon, choose the second style box from the right.
83.
Click Save to save your work.
84.
Preview your work in Internet Explorer.
Congratulations! You have created a master/detail data view page.
