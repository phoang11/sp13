Hands-On Exercise 6.1
Connecting SQL Server Data to SharePoint
Exercise list …








Objective

 

MSC manages a customer database in SQL Server. This is the best way to manage all of their customer information. Each department needs to leverage these contacts in their department site.

 

In this exercise, you will

  • Create an external content type for the Customer database table
  • Make an external list based on the external content type
Creating an external content type
1.
Open your department site in SharePoint Designer 2013 if it is not already open.
2.
Select External Content Types in the Site Objects pane.
3.
Click the External Content Type button in the New section of the ribbon.
4.
Click New external content type beside the word "Name" in the External Content Type Information section.
5.
Set the Name to [Island][Department] Customers
6.
Click New external content type beside Display Name.
7.
Change the Display Name to:
[Island]<Space>[Department]<Space>Customers
8.
Confirm the namespace is http://[island].msc.com/sites/[department]
9.
Click the pull-down menu for Office Item Type.
10.
Select Contact from the list.
11.
Make sure "Offline Sync for external list" is set to Enabled.
12.
Select Click here to discover external data sources and define operations.
This will take you to the Operations Designer page.
13.
Click the Add Connection button.
14.
Select SQL Server as the data source.
15.
Click OK.
The SQL Server Connection dialog will open.
16.

In the SQL Server Connection dialog, set the following information:

Database Server: MSCSERVER

Database Name: MSC_Company_DB
Name: [Island] [Department] Database

17.
Click the radio button next to Connect with Impersonated Custom Identity.
18.
Set the Secure Store Application ID to MSCTargetAppID
Enter the name of the Secure Store Application ID exactly as it is provided above.
19.
Click OK.
20.
Enter the following credentials when prompted for a username and password:
Username: SharePointGroupAccount
Password: Password1
The MSC Company Database appears in the Data Source Explorer. Your database connection will appear as [Island] [Department] Database.
21.
Click the plus sign to the left of [Island] [Department] Database.
22.
Click the plus sign to the left of Tables.
Warning! The first time you expand the MSC Customer Database objects, you may get a warning dialog. If so, click OK.
23.
Right-click the CurrentCustomers table.
24.
Select Create All Operations.
We are going to create five operations for create, read, update, and delete (CRUD) for the database.
25.
Click Next in the All Operations dialog.
26.
Select the BusinessEntityID if not already selected.
27.
Select the box next to Map to Identifier in the Properties pane if not already selected.
All operations require a unique identifier (key field).
28.
Clear the box next to Suffix.
29.
Select the BusinessEntityID field.
30.
Click the Show in Picker checkbox.
31.
Select the FirstName field.
32.
Change the Office Property to First Name.
33.
Click the Show in Picker checkbox.
34.
Select the LastName field.
35.
Change the Office Property to Last Name.
36.
Click the Show in Picker checkbox.
37.
Select the PhoneNumber field.
38.
Change the Office Property to Business Telephone Number.
39.
Click the Show in Picker checkbox.
40.
Select the EmailAddress field.
41.
Change the Office Property to Email 1 Address.
42.
Click the Show in Picker checkbox.
43.
Click Next.
44.
Click Add Filter Parameter on the Filter Parameters Configuration page.
Large database tables and views can cause problems for SharePoint external content types. To be on the safe side, we are going to limit the number of Customer rows returned to 250.
45.
Set the Filter Properties Data Source Element to BusinessEntityID, if not already set.
46.
Select the Click to Add link to open the Filter Configuration dialog.
47.
Set these properties in the Filter Configuration dialog:
New Filter: CustomerLimit
Filter Type: Limit
Filter Field: BusinessEntityID
48.
Click OK.
49.
Set Default Value to 250 in the Properties pane. Don't press <Enter> because the screen will close.
Online attendees: Use the number keys under the function keys instead of the keypad (keypad may not work).
50.
Click the Add Filter Parameter button on the Filter Parameters Configuration page again to add another filter to return only customers from your department.
51.
Set the Filter Properties Data Source Element to Department
52.
Select the Click to Add link to open the Filter Configuration dialog.
53.
Set these properties in the Filter Configuration dialog:
New Filter: DeptFilter
54.
Click OK.
55.
Set Default Value to [Department] in the Properties pane.
Replace [Department] with your department name.
56.
Add one more filter to return only customers from your island. Click Add Filter Parameter on the Filter Parameters Configuration page.
57.
Set the Filter Properties Data Source Element to Region
58.
Select the Click to Add link to open the Filter Configuration dialog.
59.
Set this property in the Filter Configuration dialog:
New Filter: RegionFilter
60.
In the Properties pane, set the Default Value to [Island]
Replace [Island] with your island name.
61.
Click Finish to complete the Operations creation.
62.
Click Summary View in the Views section of the ribbon.
63.
Select the BusinessEntityID from the Fields section.
64.
Click the Set as Title button in the ribbon.

External Content Types become External Lists. All lists have a required Title column. To make our SQL Server content work with a SharePoint list, it will need to have a Title field defined.

 

Occasionally, Designer will show the Set as Title field as disabled. To fix this, close and reopen your site in Designer. Then, go back into the external content type and set the title.

65.
Click Save to save your work.
Making an external list
66.
Click Lists and Libraries in the Site Objects pane.
67.
Select External List from the New group in the ribbon.
68.
In the External Content Types Picker dialog, select [Island][Department]Customers under your department site URL.
69.
Click OK.
70.
Fill in the following fields in the Create External List dialog:

Name: Customers

Description: [Island] region [department] customers

Replace [Island] with your island and [department] with your department.
71.
Click OK.
72.
Click Customers in the External Lists group.
73.
Press <F12> or click Preview in Browser in the ribbon.
That's SQL table data in SharePoint! In an upcoming exercise, you will format the data for an improved presentation.
Congratulations! You now have direct access to your department's customers from SharePoint!

This is the end of the exercise.
